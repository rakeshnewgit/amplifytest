New IT Delivery Patterns : AWS Analytics 2 - Step 2: Crawl with Glue

Created by Karina Jonane, last modified by Thomas R. Evans on May 12, 2021

In this step, you will use AWS Glue to automatically discover and structure your data so that it can be queried with analytical tools.  What is Glue?  AWS Glue is a managed ETL (extract, transform, load) service that makes it easy to prepare and load data for analytics.  Learn more about AWS Glue.    

With AWS Glue the first step is to define a crawler to populate the Glue Data Catalog with metadata table definitions. Users point the crawler at a data store, and the crawler creates table definitions in the Data Catalog. In addition to table definitions, the AWS Glue Data Catalog contains other metadata that is required to define ETL jobs.

© Amazon Web Services, Inc. 

This exercise uses the AWS Glue service to query an Amazon S3 Data Lake. In other words, you are going to create crawler - a program that connects to a data store (source or target), progresses through a prioritized list of classifiers to determine the schema for your data, and then creates metadata in the AWS Glue Data Catalog.

Tables and databases are objects containing metadata only

Tables and databases in AWS Glue are objects in the AWS Glue Data Catalog. They contain metadata; they don't contain data from a data store (which in this exercise is the data your reviewed residing in the shared S3 bucket).

Access AWS Glue

  1. Find the AWS Glue service in AWS Services list by searching for "glue" with the Quick Search function Click the returned service entry.

    If this is your first time in the Glue service, you may see a Welcome screen. Just click Get started to continue.


  2. In the AWS Glue console, find the Data Catalog entries in the left-hand navigation area and click Crawlers.

Create your Glue Crawler

The crawler connects to your data store in S3 and automatically determines its structure to create the metadata for your table.

  • Click Add crawler.

  • Step through the Add Crawler wizard as detailed below:



    Add information about your crawler:
     Name your crawler so that you can identify it when referencing it later.
    You can leave Description and classifiers (optional) unmodified. Click     Next.
    Add a data store:

    Select Data store as the Crawler source type.
    Select Crawl all folders as the Repeat crawls setting and click Next.

        

    Confirm that Data store is set to S3. Confirm that Crawl data in is set to Specified path in my account.

     Populate Include path as follows: • Click on the folder icon to choose the S3 path to the file to be crawled.

    Note: if you do not see a selection dialog after clicking the folder icon, try a different web browser.

     You will be crawling a shared test file located in an S3 bucket specific to the AWS account you are using

    ::: if you are using the cmt-cloud-native-001 account expand the cmt-cn-glue bucket path

    ::: if you are using the cmt-cloud-native-002 account expand the cmt-cn-002-glue bucket path

    ::: if you are using the cmt-cloud-native-003 account expand the cmt-cn-003-glue bucket path


    • tick the radio button next to Product-Marketing-data.
    Note! Select only the folder holding the file you intend to have crawled. Do not select the file itself.

     When the relevant folder is selected, click the Select button below the window to see your selected S3 path populated.


     Ignore the Exclude patterns option. Click Next.

    Add another data store:
     Select No to the option of adding another data store and click Next.
    IAM Role:

    Select the Choose an existing IAM role option. and select the IAM role 'AWSanalytics2'.

    Double-check your selected IAM role!

    There are multiple IAM roles to choose from...Double-check to confirm you've selected the AWSanalytics2 IAM role.


    Click     Next.

    Schedule:

    Confirm that Frequency is set to Run on demand. Click Next.
    Configure the crawler's Output:

    Click the Add database button, The Add database interface will drop into view
    .
    Give your database a name you will recognize. Ignore the optional Description and location. Click Create.

    The Add database interface will close.

    Leave the Prefix added to tables field blank.
    Leave Configuration options at their default values then click Next.

    Final Review:

    Review to confirm the crawler configuration, (You can step back in the wizard to make corrections as necessary.)

    Click     Finish to create the crawler.

Run the crawler

From the crawlers list, you will  have the option to run the crawler you created. Tick the check box next to your crawler and click the Run crawler button at the top of the crawler list.

  • The crawler will go from Starting status to Stopping to Ready in the initial Run process.


Expect the crawler to take +/- 90 seconds to show Ready status. Do not be concerned that the status reports 'Stopping' during this process. This is expected behavior.

You can refresh the status output by clicking on the refresh link in the upper-right corner of the crawler listing.

You will see a status similar to the following when the crawler start-up process is complete.

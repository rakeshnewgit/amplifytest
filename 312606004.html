Visualize your data: Demographic – education level

The example below will result in a bar chart intending to show the Yes/No responses across an education level distribution.

Choose your Visual type

(Visual types are organized below the field list.)

  • Select the Vertical stacked bar chart visual type,

Populate your field wells

Note: If you can’t see the Field wells parameters, click on the side double arrows to expand the interface.


Note: among the data attributes

(You can either drag the field name from the field list and drop it on the target well, or select it from the field well's drop-down.)

  • Add y from the field list to the x axis field well.

  • Add education from the field list to the Group/Color field well.
  • (Leave the Value field well blank for now)

Result

If you mouse over each layer in the stacks, you'll see it reporting raw numbers of responses, and that's not very useful. Also, the much greater number of 'no' customers over 'yes' customers makes it difficult to tell if there is an education-level pattern in the data...

Quicksight offers an easy way to adjust for that.

Change the visual type to 100% bar chart

  • Select the Vertical stacked 100% bar chart visual type,

Upon selection of the alternate visual type, QuickSight will redraw automatically.

Result


This is better. Now when you mouse-over each education level in the responses, you see that education level's percentage representation for the particular response.

The education demographic looks pretty evenly distributed. There may be a pattern here, but certainly nothing dramatic.

Consider other demographics

Note: the field wells interface feature may minimize depending on various actions you take in QuickSight. To access it, simply toggle its expand collapse feature.


(The same applies to any other feature in the visualization UI.)

QuickSight makes it very easy to experiment. You can try one or more of the following and see the results immediately. For example, you could

  • Change to job (or any other available grouping) from education

  • Or instead add job (or any other available grouping) as a drill down layer

Visualize your data: Value – consumer confidence index

If demographic views don't shed a lot of light on our data, perhaps some of the environmental variables we have access to will.

The example below will result in a Heat Map intending to show – per advertising campaign – the number of successful contact attempts relative to the consumer confidence index value.

Optional: Add another visual

  • If you want to retain the visualization you've been working on in this analysis, choose Add visual before proceeding

Choose your Visual type.

(Visual types are organized below the field list.)

  • Select the Heat Map visual type.

Populate your field wells.

(You can either drag the field name from the field list and drop it on the target well, or select it from the field well's drop-down.)

  • Add campaign from the field list to the Rows field well.

  • Add cons_conf_idx from the field list to the Columns field well.
  • Add poutcome from the field list to the Values field well.

Filter out non-successful outcomes.

  • Click to select the filter function, then click to create a filter.
  • Select poutcome as the filtered field.
  • Click on Include All modify the values to be reflected, then tick the success check box.
  • Click Apply to apply the filter (at the very bottom of the filter panel).


Take a few minutes to explore other filters and visual types!

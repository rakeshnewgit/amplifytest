New IT Delivery Patterns : AWS Analytics 2 - Step 1: Review S3 Data

Created by Karina Jonane, last modified by Thomas R. Evans on Apr 14, 2021


What is S3?  Amazon S3 is AWS' Simple Storage Service that is built to store and retrieve any amount of data with virtually unlimited scalability.  S3 is used to store data for leading companies in virtually every industry including examples like Netflix.

Learn more about Amazon's S3.

Getting familiar with S3

While this step does not actually involve loading data for the ad hoc analysis covered in this exercise – you will be using data already in S3 – you should follow the steps below as a way to become familiar with standard S3 bucket organization. Getting familiar with S3 now will make it easy to run through other exercises in this space, including ones that involve analysis of data you load into S3 yourself to leverage Glue and Athena, as well as Amazon Machine Learning.

S3 Storage

  1. Once you are signed into AWS, you will see all available AWS services. From that list, search for S3 in the quick search field and select the top-returning entry.

  2. Once in the Amazon S3 Console, search for the relevant bucket (differs per account – see illustration below) and click to navigate into its contents.

    If you are using the cmt-cloud-native-001 account:
    search for cmt-cn-glue

    If you are using the cmt-cloud-native-002 account:
    search for cmt-cn-002-glue

    If you are using the cmt-cloud-native-003 account:
    search for cmt-cn-003-glue

  3. In the bucket, you will see one or more folders. The Product-Marketing-data folder holds a test file that you can use as your test data for this quick exercise.

Review Data

  1. Download data to the desktop.
    Tick the check box next to the prodmarketing.csv file, then choose Download from the Actions menu.




    Different web browsers handle file downloads differently. Save the file to your workstation. Make note of its location, then browse to it in your system's file explorer.


  2. Open data.
    Open the file with MS Excel - double click the file.


  3. Explore data.
    This dataset contains product marketing information: the historical data for customers who have been offered products similar to a bank term deposit. For this exercise, we are using a formatted dataset from UCI Machine Learning Repository via AWS.

    prodmarketing.csv is a dataset with 41,189 records.  Each record has 20 parameters and one binary attribute.

    Demographics:

    • age: Customer's age (numeric)
    • job: Type of job (categorical: 'admin.', 'services', ...)
    • marital: Marital status (categorical: 'married', 'single', ...)
    • education: Level of education (categorical: 'basic.4y', 'high.school', ...)

    Past customer events:

    • default: Has credit in default? (categorical: 'no', 'unknown', ...)
    • housing: Has housing loan? (categorical: 'no', 'yes', ...)
    • loan: Has personal loan? (categorical: 'no', 'yes', ...)

    Past direct marketing contacts:

    • contact: Contact communication type (categorical: 'cellular', 'telephone', ...)
    • month: Last contact month of year (categorical: 'may', 'nov', ...)
    • day_of_week: Last contact day of the week (categorical: 'mon', 'fri', ...)
    • duration: Last contact duration, in seconds (numeric). Important note: If duration = 0 then y = 'no'.

    Campaign information:

    • campaign: Number of contacts performed during this campaign and for this client (numeric, includes last contact)
    • pdays: Number of days that passed by after the client was last contacted from a previous campaign (numeric)
    • previous: Number of contacts performed before this campaign and for this client (numeric)
    • poutcome: Outcome of the previous marketing campaign (categorical: 'nonexistent','success', ...)

    External environment factors:

    • emp.var.rate: Employment variation rate - quarterly indicator (numeric)
    • cons.price.idx: Consumer price index - monthly indicator (numeric)
    • cons.conf.idx: Consumer confidence index - monthly indicator (numeric)
    • euribor3m: Euribor 3 month rate - daily indicator (numeric)
    • nr.employed: Number of employees - quarterly indicator (numeric)

    Attribute:

    • y: Has the client subscribed a term deposit? (binary: 'yes','no')
